## requirement

> 生成一个专门分析用户需求并且制定计划编写前端代码的流程

```txt
目前市面上的低代码工作流项目，有什么区别？我发现无论是哪种工作流Agent这些都有些虚假，你觉得呢？
预期收益矩阵这块我觉得：1. 技术外包这块，需要吃技术经验这些的，普通人一般不会给你发单子。2. API订阅这块感觉市面上很多平台都有了，我们能提供哪些好的API服务呢？ 3. 内容付费这块我觉得还是可以的，毕竟课程、社群、模板啥的还是不错的。模板这块我觉得可以做几个整个完整的工作流来组合进行一个项目的完整开发。 4. 平台补贴这块，考虑的可能会偏制定内容一些不太做的来。

其次大家对于论文这块需求也是挺大的，我觉得可以围绕大模型工作流Agent，来制定一系列的流程，开发对应领域的【Agent工作室】来做对应的任务。我看到了整个市场上，大多数生产服务例如 v0.dev, bolt, replit.这些都是做页面+前端的，即使做后端也是小部分，且功能似乎不是那么如意的。似乎没有全流程开发App这块的，没有全流程部署 前后端、全流程利用资源上线 API 服务的，我觉得这些是个开创的新方向。

似乎大家都有对 PDF 英文翻译这块需求，这个可以考虑、采用LLM大模型API的文档识别来做。
```
